# 角色设定：高级 Python & Streamlit 数据开发工程师

## 背景
你正在维护 `finance_ai_system` 项目。具体任务是增强 `modules/module1/mod1_table_general_processor.py` 文件。
目标是添加一个“自定义处理”功能模块，允许用户通过动态 UI 进行复杂的数据操作，而无需编写代码。该模块需支持多 Sheet/多文件处理，并具备实时预览和导出功能。

## 任务描述
请在 `mod1_table_general_processor.py` 中实现一个核心函数 `render_custom_processing(df, uploaded_files=None)`。该函数必须包含以下六大逻辑模块。**所有模块均为可选**，若用户未配置则跳过；若配置了多个功能，需在最终结果中按顺序应用，且不同功能的输出可分 Sheet 展示。

## 详细功能需求

### 1. 识别与选择处理对象 (Sheet/File Selection)
- **UI 组件**：下拉框（多选）。
- **逻辑**：
  - 默认选中第一个 Sheet。
  - 若用户上传了多个文件或 Excel 含多个 Sheet，需列出所有可用选项（格式建议：`文件名-Sheet名`）。
  - **联动逻辑**：若选择了多个 Sheet 或涉及多个文件，自动激活“跨表/跨文件操作”模块（见第 6 点）。

### 2. 动态筛选字段 (Dynamic Filtering)
- **UI 组件**：
  - 默认显示一组筛选器：[筛选字段下拉框] + [筛选值多选下拉框]。
  - 提供“+”按钮，点击动态增加新的筛选行。
- **逻辑**：
  - **筛选字段**：下拉框内容应动态读取当前数据的列名。
  - **筛选值**：根据所选字段，动态加载该列的去重全量值供多选。
  - **后端处理**：使用 Pandas 的 `isin` 或布尔索引进行过滤，支持多条件“与”关系。

### 3. 筛选后通用操作 (Post-Filter Operations)
- **UI 组件**：
  - 默认显示一组操作配置：[操作类型下拉框] + [操作字段多选下拉框]。
  - 提供“+”按钮动态新增。
- **操作类型定义**：
  - **聚合类**：求和、求平均、计数。
  - **排序类**：升序、降序。
  - **清洗类**：删除重复值、删除筛选行、保留筛选行。
  - **拼接类**：使用 `-` 或 `,` 拼接多个选定字段的文本内容。
- **输出规范**：若涉及多列操作，新生成的列头格式为 `操作名-原字段名`。

### 4. Excel 公式模拟引擎 (Excel Formula Simulator)
此模块最为复杂，需在前端模拟 Excel 公式输入体验，在后端用 Pandas 实现对应逻辑。
- **UI 结构**：
  - **内容 1（公式选择）**：下拉框，按首字母排序，包含 `SUMIFS`, `AVERAGE`, `XLOOKUP`, `FILTER`, `TEXTJOIN` 等常用办公公式。
  - **内容 2（公式说明）**：根据选择，动态显示公式语法及中文功能简述（只读文本）。
  - **内容 3（参数配置）**：根据所选公式动态渲染输入项。
    - *数值/文本参数*：支持手动输入（提示字符串用引号包裹）或下拉选择 `Sheet名-列名`（取当前行值）。
    - *区域参数*：下拉选择 `Sheet名-列名`。
    - *特殊参数*：如 XLOOKUP 的匹配模式（0/1/-1/2）、搜索模式等，需提供专用下拉框。
- **逻辑实现**：
  - 不要真的调用 Excel 引擎，而是将公式逻辑转换为 Pandas 操作（例如 `SUMIFS` 转换为 `groupby` 或条件求和，`XLOOKUP` 转换为 `merge` 或 `map`）。
  - 新列命名格式：`公式名-首个参数字段名`。

### 5. 数据透视表 (Pivot Table)
- **UI 组件**：
  - 筛选条件（动态新增）：字段 + 值。
  - 行标题（动态新增）：字段选择。
  - 列标题（动态新增）：字段选择。
  - 值字段（动态新增）：字段 + 聚合方式（求和/计数等）。
- **逻辑**：调用 `pd.pivot_table`，支持多级索引。

### 6. 跨 Sheet/跨文件操作 (Cross-Sheet/File Operations)
- **触发条件**：当第 1 步中选择了多个 Sheet 或上传了多个文件时显示。
- **UI 组件**：下拉框选择操作类型。
- **操作类型**：
  - **拼接/合并**：纵向堆叠 (`pd.concat`) 或横向连接 (`pd.merge`)。
  - **跨表公式**：引用其他 Sheet 的数据进行计算。

## 交互与输出要求
1. **实时预览**：每个功能模块配置完成后，下方应立即显示处理后的 DataFrame 前 N 行预览（使用 `st.dataframe`）。
2. **结果整合**：
   - 若仅处理单 Sheet，直接返回结果 DataFrame。
   - 若涉及多 Sheet 或多功能并行，返回一个字典 `{ "Sheet名_功能名": DataFrame }`。
3. **导出功能**：
   - 提供“分别导出”按钮：下载单个 CSV/Excel。
   - 提供“打包导出”按钮：将所有结果写入同一个 Excel 文件的不同 Sheet 中并下载。

## 代码风格要求
- 使用 Streamlit 的 `st.session_state` 管理动态增加的表单行（避免刷新页面后表单重置）。
- 封装独立的辅助函数处理 Pandas 逻辑，保持主渲染函数整洁。
- 对复杂的公式逻辑添加详细注释。
